## 🔑 KEY FEATURES

### Profile Management
- Display email (read-only)
- Edit full name
- View account status and timestamps
- Real-time updates

### Account Preferences
- **Languages:** 11 options (en, es, fr, de, it, pt, nl, pl, ru, ja, zh)
- **Voices:** 6 options (alloy, echo, fable, onyx, nova, shimmer)
- **Timezones:** 11 major timezones
- **Currencies:** 6 options (USD, EUR, GBP, JPY, CAD, AUD)

### Email Notifications
- Email notifications on/off
- Usage alerts on/off
- Marketing emails on/off
- Individual toggle per setting

### Security
- Password change with verification
- Current password required
- Minimum 8 characters
- Show/hide password toggle
- OAuth users blocked from password change

### API Key Management
- Generate secure API keys (`sk_live_{64-hex}`)
- Maximum 5 active keys per user
- Full key shown only once
- List with masked keys (last 4 chars)
- Individual revocation
- Timestamps: created_at, last_used_at

### Account Deletion
- Soft delete (preserves data)
- Password verification required
- Must type "DELETE" to confirm
- Immediate access revocation
- Full activity logging
- OAuth users must delete through provider

---

## 🔒 SECURITY MEASURES

1. **Password Security**
   - Bcrypt hashing (12 rounds)
   - Current password verification
   - Never logged or returned in responses
   - Minimum length enforcement

2. **API Key Security**
   - Cryptographically secure generation (`secrets.token_hex`)
   - Stored as bcrypt hash (never plaintext)
   - Only last 4 chars visible after generation
   - Rate limiting (5 keys max)
   - Individual revocation

3. **Account Deletion**
   - Password verification required
   - Explicit confirmation ("DELETE")
   - Soft delete (audit trail)
   - Activity logging with metadata
   - Cannot be reversed

4. **Authorization**
   - All endpoints require authentication
   - Users can only access own data
   - JWT token validation
   - Automatic token refresh

## 📊 OBSERVABILITY

### Prometheus Metrics
```python
sonoro_profile_updates_total{user_id, status}
sonoro_password_changes_total{user_id, status}
sonoro_api_keys_generated_total{user_id}
sonoro_api_keys_revoked_total{user_id, key_id}
sonoro_account_deletions_total{user_id, email}
```

### Activity Logging
All operations logged to `account_activity` table:
- Activity type
- User ID
- IP address
- User agent
- Metadata (changes, IDs, etc.)
- Timestamp

### Monitoring Queries
```promql
# Profile updates per hour
rate(sonoro_profile_updates_total[1h])

# API key generation rate
rate(sonoro_api_keys_generated_total[1h])

# Failed password changes
rate(sonoro_password_changes_total{status="error"}[1h])

# Account deletion trend
increase(sonoro_account_deletions_total[7d])
```


```bash
# Check api_keys table
docker-compose exec postgres psql -U sonoro -d sonoro -c "\d api_keys"
```

## 🔄 ROLLBACK PLAN

If issues arise, rollback is simple:

```bash
# Step 1: Downgrade database
docker-compose exec api alembic downgrade -1

# Step 2: Restart services
docker-compose restart api frontend

# Step 3: Verify
docker-compose exec postgres psql -U sonoro -d sonoro \
  -c "SELECT * FROM information_schema.tables WHERE table_name='api_keys';"
# Should return 0 rows
```

**Rollback Time:** ~1 minute  
**Data Loss:** API keys will be lost (soft-deleted accounts remain)
